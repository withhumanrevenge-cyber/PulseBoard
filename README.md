# PulseBoard: The Reputation Protocol

PulseBoard is a high-velocity metrics directory designed to automate professional transparency. It doesn’t just show what you built; it tracks the **Reputation of Shipping.**

Every node you connect—from GitHub to specialized stack registries—updates your persona in real-time, providing a verified, animatic summary of your technical velocity.

### Navigation Protocol
*   **Directory**: Explore the global shipping nodes. Sort by language, frequency, and impact.
*   **Persona Profile**: An editorial-grade summary of your artifact history, real-time contributions, and verified top-language mapping.
*   **The Console**: Synchronize your private stack. Manage identity privacy and registry connections in one node.

### Core Stack
*   **Runtime**: Next.js 16 (Turbopack Deployment)
*   **Motion Architecture**: Framer Motion (Spring Physics)
*   **Identity Identity**: Clerk (SmartAuth Boundary)
*   **Registry Storage**: Supabase (Real-time SQL Nodes)
*   **Data Bridge**: Octokit (GraphQL/REST GitHub Sync)

### Launch Sequence
1.  **Connect Registry**: Populate `env.local` with Clerk, Supabase, and GitHub tokens.
2.  **Ignite Pulse**: `npm run dev`
3.  **Ship**: Populate your dashboard and launch your public persona node.

*Made for those who ship.*
